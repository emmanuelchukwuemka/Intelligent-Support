# Intelligent Support Framework for Stress Management

A full-stack Decision Support System + AI stress management platform:
DASS-21-style assessment → Random Forest severity classifier →
collaborative-filtering intervention recommendations → progress tracking
→ knowledge base → admin panel.

---

## Deploying to Render + Neon

**Database**: [Neon](https://neon.tech) (serverless PostgreSQL — free tier available)
**Hosting**: [Render](https://render.com) (Python web service — free tier available)

### Step 1 — Create a Neon database

1. Sign up at **https://neon.tech** → create a new project
2. Neon creates a default database. Go to your project **Dashboard →
   Connection Details**
3. Copy the **connection string** — it looks like:
   ```
   postgresql://user:password@ep-xxx-xxx.us-east-2.aws.neon.tech/neondb?sslmode=require
   ```
   Keep this — you'll paste it into Render in Step 3.

### Step 2 — Push to GitHub

```bash
# Create a new GitHub repo at github.com, then:
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git push -u origin master
```

### Step 3 — Deploy on Render

1. Go to **https://render.com** → **New → Blueprint**
2. Connect your GitHub repo — Render reads `render.yaml` and creates a
   **stress-support** Python web service
3. In the service's **Environment** tab, set these variables:

   | Key | Value |
   |---|---|
   | `DATABASE_URL` | Your Neon connection string from Step 1 |
   | `ADMIN_PASSWORD` | A strong password for the admin account |

   `JWT_SECRET` is auto-generated by Render.

4. Click **Apply** — Render will:
   - Install Python dependencies
   - Train the Random Forest model (`python ml/train_model.py`, ~60 s)
   - Run `python init_db.py` to create tables and seed data in your Neon DB
   - Start Gunicorn

5. Open your Render URL. Admin login:
   - Username: `admin`
   - Password: the `ADMIN_PASSWORD` you set (default `Admin@1234`)

### What Render builds and runs

```
Build:       pip install -r requirements.txt && python ml/train_model.py
Pre-deploy:  python init_db.py    ← idempotent; safe on every re-deploy
Start:       gunicorn app:app --workers 2 --bind 0.0.0.0:$PORT --timeout 120
```

The frontend (HTML/CSS/JS) is served directly by Flask on the same URL —
no separate static site or CDN needed.

> **Cold starts**: Render's free tier spins down after 15 min idle;
> the first request after that takes ~30 s. Upgrade to a paid instance
> for always-on. Neon's free tier also suspends after inactivity but
> resumes in milliseconds.

## Local development

Two options:

### Option A — single server (matches production)

```bash
cd backend
python -m venv venv
venv/Scripts/pip install -r requirements.txt   # Windows
# source venv/bin/activate && pip install ...   # Linux/Mac

# Create and seed the local DB (PostgreSQL must be running):
psql -U postgres -h 127.0.0.1 -c "CREATE DATABASE stress_support;"
python init_db.py          # creates tables, seeds data, creates admin

# Train the ML model:
python ml/train_model.py

# Start:
python app.py
```

Open **http://127.0.0.1:5000**

### Option B — two servers (API on :5000, static on :8000)

```bash
# Terminal 1
cd backend && python app.py          # Flask API on :5000

# Terminal 2
cd frontend && python -m http.server 8000
```

Open **http://127.0.0.1:8000** — `config.js` detects port 8000 and
automatically points at `http://127.0.0.1:5000/api`.

## Environment variables

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Render only | Injected automatically from the DB service |
| `JWT_SECRET` | Yes | Auto-generated by Render; set manually for local |
| `ADMIN_PASSWORD` | Optional | Password for the `admin` account (default `Admin@1234`) |
| `ADMIN_USERNAME` | Optional | Admin username (default `admin`) |
| `ADMIN_EMAIL` | Optional | Admin email (default `admin@stress-support.local`) |
| `PORT` | Render only | Injected by Render; Gunicorn binds to it |

For local dev create `backend/.env` (already in `.gitignore`).

**Using a local Postgres:**
```env
DB_HOST=127.0.0.1
DB_PORT=5432
DB_NAME=stress_support
DB_USER=postgres
DB_PASSWORD=
JWT_SECRET=any-local-secret
```

**Using Neon locally** (paste the connection string):
```env
DATABASE_URL=postgresql://user:password@ep-xxx.us-east-2.aws.neon.tech/neondb?sslmode=require
JWT_SECRET=any-local-secret
```

## Notes

- **Model**: excluded from git (`model.joblib`). Render retrains it
  during every build (`python ml/train_model.py`, ~60 s). Test accuracy
  ≈ 76 % — intentional label noise simulates real-world uncertainty.
- **Database**: `init_db.py` is idempotent — safe to run on every
  re-deploy without touching existing user data (`CREATE TABLE IF NOT
  EXISTS`, `INSERT ... ON CONFLICT DO NOTHING`).
- **Cold starts**: Render's free plan spins down after 15 min idle;
  first request after that takes ~30 s. Upgrade to a paid instance for
  always-on.
